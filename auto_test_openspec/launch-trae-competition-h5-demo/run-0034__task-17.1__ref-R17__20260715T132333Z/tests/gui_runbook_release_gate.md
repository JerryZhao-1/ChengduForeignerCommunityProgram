# MCP-only release-gate GUI runbook (R17 run-0034)

Use the configured browser MCP against the URL printed by the start command below. Do not create or execute Playwright/Selenium/browser scripts.

## Start command (Supervisor executes)

```bash
env VITE_API_MODE=mock pnpm --filter @community-map/mobile dev:h5 --host 127.0.0.1 --port 5174
```

URL: `http://127.0.0.1:5174/?guest=judge#/pages/onboarding/welcome`

Run the full flow for four representative profiles in both zh and en (8 runs total). For each profile, time welcome → complete and confirm it is under 180 seconds. This matrix is identical to run-0033's `tests/gui_runbook_judge_journey.md` and is referenced here for completeness.

## Release-gate-specific assertion points

In addition to run-0033's judge-journey assertions, this runbook explicitly verifies the four GUI gates requested for the AI-free release candidate:

### 1. No browser console errors
- Open browser DevTools console before navigating.
- Complete the full welcome → complete flow for at least one profile in each locale.
- Assert: zero `console.error` entries during the entire flow. Warnings are acceptable; errors are not.
- Record any console output in `logs/gui-screenshot-index.md`.

### 2. No horizontal scroll at 390px
- Set viewport to 390 × 844.
- Navigate through welcome → preferences → plan → route-map → demo-confirm → complete.
- For each page, assert: `document.documentElement.scrollWidth <= 390` (no horizontal overflow).
- Capture `outputs/zh-390px-no-horizontal-scroll.png` and `outputs/en-390px-no-horizontal-scroll.png`.

### 3. No raw enum leakage
- Throughout the flow, verify that no raw enum values appear in rendered UI text. Specifically check:
  - Preference options show localized labels (e.g., "社区服务 / Community service"), not `community-service`.
  - Plan reasons show localized dimension text, not `primary_interest` / `arrival_context` / `household_type` / `accessibility_need` keys.
  - Error states (if triggered) show localized error copy, not `VALIDATION_ERROR` / `RATE_LIMITED` codes.
  - Delivery mode badge shows localized "离线 / Offline" or "在线 / Online", not `offline` / `online`.
- Capture `outputs/zh-plan-no-raw-enum.png` and `outputs/en-plan-no-raw-enum.png`.

### 4. No AI-generated text
- Throughout the flow, verify that no AI/LLM/model-related text appears anywhere in the rendered UI. Forbidden strings include:
  - "generated by AI" / "AI 生成" / "AI-generated"
  - "model status" / "模型状态"
  - "powered by LLM" / "powered by GPT" / "powered by DeepSeek"
  - "generated_by" / "ai_status" / "usage" / "prompt"
  - Any reference to a model name, provider, or generation source
- The plan page should show only the curated editorial disclosure (e.g., "由桐梓林社区策展目录生成 / Generated from the Tongzilin community curated catalog").
- Capture `outputs/zh-plan-no-ai-text.png` and `outputs/en-plan-no-ai-text.png`.

### 5. zh/en switch preserves session
- Complete the flow in zh, reach the plan page.
- Switch to en.
- Assert: the plan is not re-matched, `scenario_key` is unchanged (if visible in DevTools network/state), all copy updates immediately to English.
- Switch back to zh.
- Assert: copy returns to Chinese, plan unchanged.
- Capture `outputs/zh-en-switch-plan-unchanged.png`.

### 6. route-map renders both stops
- On the plan page, click "View route" / "查看路线".
- Assert: route-map renders BOTH stops in order:
  - Stop 1: place card (cover, localized name, coordinates, "View place" / "查看地点" button)
  - Stop 2: event card (cover, localized title/summary, "View event" / "查看活动" button)
- Assert: the "map enhancement unavailable" / "地图增强不可用" notice is shown and the route list is fully usable without a map.
- Capture `outputs/zh-route-map-two-stops.png` and `outputs/en-route-map-two-stops.png`.

### 7. Completion state
- Click "Mark visited" / "标记已到访" on the place card.
- Click "Demo Confirm" / "演示确认" on the event card.
- Assert: "Finish Route" / "完成路线" becomes enabled only after both actions.
- Click "Finish Route".
- Assert: complete page shows place 1/1 and event 1/1.
- Capture `outputs/zh-complete.png` and `outputs/en-complete.png`.

### 8. Desktop layout (1280 × 900)
- Set viewport to 1280 × 900.
- Repeat one profile.
- Assert: layout is centered with max-width, no horizontal scroll, targets remain ≥44px with visible focus.
- Capture `outputs/zh-desktop-plan.png`.
