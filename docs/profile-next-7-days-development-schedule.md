## 建议积分规则

| 行为 | 积分 | 说明 |
| --- | ---: | --- |
| 首次完善 Profile | +20 | 一次性奖励。 |
| 报名活动成功 | +10 | 每个活动只奖励一次。 |
| 收藏帖子/Event/地点 | +2 | 每个内容只奖励首次收藏。 |
| 发表评论 | +5 | 每条有效评论奖励一次。 |
| 达成首次报名里程碑 | +30 | 里程碑奖励。 |
| 达成 5 次互动里程碑 | +50 | 互动包括报名、收藏、评论。 |

## 每日验证清单

每天结束前至少完成：

```bash
corepack pnpm --filter @community-map/mobile typecheck
corepack pnpm --filter @community-map/mobile build:mp-weixin
```

涉及 shared mock、API contract 或测试时额外运行：

```bash
corepack pnpm test
```

## 风险与缓冲

- 入场券信息目前依赖 mock ticket 数据，后续接真实 API 时需要统一 ticket contract。
- 积分系统容易扩展过大，7 天内建议先做可体验规则，不做复杂风控。
- 里程碑奖励要避免重复发放，需要在 mock 数据里记录已领取/已达成状态。
- 设置功能先聚合已有入口，不急于新增真实账号安全、隐私、上传能力。

- Profile 能展示用户资料、关注/粉丝、收藏/评价、~~入场券~~（已完成）、设置入口、积分与里程碑。
- ~~报名成功后的入场券能在 Profile 中查看。~~（已完成）
