# 选定方向：First 120 Minutes 社区策展匹配

## 结论

比赛版选择“有限画像 + 版本化策展目录 + 可解释两小时路线”。产品运行时不调用 AI/LLM。

## 核心机制

- 五个必填字段，其中语言控制展示，其余四维组成 576 个逻辑场景。
- 每个场景有唯一稳定 `scenario_key`，同时保存中英文摘要和四条选择理由。
- 结果固定为一个地点加一个演示活动，共 120 分钟。
- 在线 API 与离线 H5 调用同一 shared matcher 和 `tongzilin-curated-v1` 目录。
- 参与需求只改变安全确认建议，不宣称未经核实的地点设施。

## 为什么适合比赛

- 评委可在 2–3 分钟内走完完整价值闭环。
- 576/1,152 和 API/local parity 都可程序化验证，不依赖主观演示。
- 无登录、无模型、无外部叙述服务依赖，公开 H5 与离线 ZIP 都可稳定体验。
- 产品边界透明：Demo Confirm 不代表真实报名，无障碍提示不代表设施认证。

## 被放弃方向

- 纯功能陈列：缺少清晰的用户结果和主叙事。
- 对话助手或模型叙述增强：增加不稳定外部依赖，也无法替代社区信息核验。本次比赛不采用。

历史 R1–R9 可能保留早期探索截图；当前实现与放行条件只以 R10–R18 为准。
