# 桐邻 First 120 Minutes：比赛 H5 设计规格

## 1. 体验目标

- 评委 10 秒内理解产品价值，30 秒内完成画像，180 秒内走完闭环。
- 用户能看懂四个选择如何影响方案，而不是面对技术来源或黑盒状态。
- 首屏计划区域同时给出 120 分钟目标、策展摘要、第一行动和路线入口。
- 地图、网络和图片增强失败时，核心路线仍可完成。

## 2. 信息架构

1. Welcome：价值主张、访客透明说明、普通体验与示例入口。
2. Preferences：语言、首要兴趣、到达阶段、家庭结构、参与需求。
3. Plan：摘要、透明说明、四维解释、地点与活动、完成操作。
4. Route：地图为增强项，结构化路线清单为基础能力。
5. Complete：地点 1/1、演示活动 1/1 和 Start Over。

兴趣与参与需求均为单选。`none` 必须显示为“无额外需求 / No additional need”，所有四个业务维度都选择后才能提交。

## 3. 视觉方向

延续现有 Organic Community Field Guide，不引入新 UI 库：

- 页面底色 `#F6F0E5`，卡片 `#FBF8F1`。
- 主色 `#0F766E`，深色文字 `#263331`，次要文字 `#5C6B68`。
- 活动操作使用 `#E66A45`；安全/参与提示使用 `#D39A3A`。
- 标题使用 Fraunces / Songti SC，正文使用系统无衬线字体。
- 桌面居中，移动端重点验证 390px；不锁死固定画布。
- 卡片以浅边框和空间层级为主，阴影保持克制。

这些 token 是既有比赛 H5 设计系统 override；小程序端新增 UI 仍优先采用 TDesign MiniProgram。

## 4. 交互规格

- 所有点击区域至少 44px。
- 单选卡片提供 `role=radio`、`aria-checked`、键盘 Enter 和可见焦点。
- 新选择自动替换旧选择，不出现多选或重复值。
- 加载步骤依次为：核对时间偏好、匹配社区地点、整理参与提示、准备两小时路线。
- 在线成功不显示来源 badge。
- 离线只显示“离线演示 · 使用同版本本地社区目录”。
- Demo Confirm 必须伴随“不创建真实报名”的说明。
- Finish Route 只有在地点到访和活动演示确认完成后可用；地点 404 可标记不可用并调整分母。

## 5. 计划页层级

1. 页面标题与总时长。
2. “基于桐梓林社区策展信息匹配”的非技术说明。
3. 策展摘要。
4. 第一行动与路线按钮。
5. “为什么这样匹配”：严格按首要兴趣、到达阶段、家庭结构、参与提示展示四条理由。
6. 地点与活动卡片。
7. Finish Route。

四维解释不显示 raw enum、内部 key、模型状态或服务器原始错误。

## 6. 双语规则

- `zh` / `en` catalog 递归 key 完全一致。
- 方案同时持有中英文内容，当前 locale 只决定渲染语言。
- 切换语言不重新匹配、不修改 `scenario_key`、不清空 plan/session。
- 所有 1,152 个语言渲染案例必须有非空摘要、理由、标题和提示。

## 7. 无障碍与诚实表达

- 颜色对比至少满足 WCAG AA；不能只靠颜色表示选中或完成。
- 轮椅、低视力、行动较慢、听力支持和安静环境只生成出发前确认/陪同建议。
- 不得写“无障碍卫生间”“有坡道”“配备助听设备”“环境安静”等未经目录认证的地点事实。
- 图片失败显示占位或保持文本内容；地图失败仍显示路线清单。

## 8. 响应式与跨端

- H5：390px 手机视口完整可用，桌面 `max-width` 居中。
- `scroll-view` 内容可滚动，不依赖固定高度。
- H5 专用 DOM 和比赛入口使用编译条件保护。
- `build:mp-weixin` 必须通过，且不新增比赛入口到既定小程序信息架构。

## 9. 禁止项

- 产品文案声称模型生成、实时容量或真实报名成功。
- 显示生成来源、模型状态、token 或 raw enum。
- 用离线状态改变策展内容。
- 引入新 UI 组件库或大规模重做现有应用。
- 使用插画、emoji 或未经授权/核验的图片和设施声明。

## 10. 验证视口与截图

- 390px 中文兴趣单选。
- 390px 英文参与需求单选。
- 390px 完整计划页与四条解释。
- 离线 badge。
- 桌面居中计划页。
- 键盘焦点状态。
- H5 与 mp-weixin 构建结果。
