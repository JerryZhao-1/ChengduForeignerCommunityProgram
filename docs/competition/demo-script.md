# 桐邻 First 120 Minutes：2–3 分钟演示脚本

## 演示目标

用一条最强路径说明：新居民从“不知道社区里有什么”，到获得一条有明确理由、可以立即执行的 120 分钟融入路线。

## 演示前准备

- 主入口使用公开移动端 H5；不登录、不扫码、不输入验证码。
- 浏览器视口优先使用 390px 宽度。
- 准备在线 API 和断网/阻断 API 两种状态。
- 不进入真实活动报名；只使用 Demo Confirm。

## 时间轴

### 0:00–0:20｜问题与入口

打开 Welcome 页，说明刚搬到桐梓林的人常常知道“附近有很多资源”，却不知道今天先去哪、怎样参与、需要确认什么。

点击“为我制定计划”。强调访客模式无需登录或微信授权。

### 0:20–0:55｜四维单选画像

依次选择：

1. 中文；
2. 社区服务；
3. 初来第一周、独居；
4. 无额外需求。

展示兴趣和参与需求均为单选，新选择会替换旧选择。提交后加载状态依次显示核对时间、匹配地点、整理提示、准备两小时路线。

### 0:55–1:35｜可解释策展方案

方案页首屏展示：

- 120 分钟目标；
- 社区策展匹配摘要；
- 第一行动；
- 路线入口。

向下展示“为什么这样匹配”，指出四条理由分别对应首要兴趣、到达阶段、家庭结构和参与提示。说明系统预写 21 个双语维度模块，并确定性组合为 576 个稳定场景和 1,152 个本地化案例，而不是临时生成文案。

### 1:35–2:15｜参与闭环

打开路线清单和地点详情，返回后标记地点已到访；对活动点击 Demo Confirm。明确该动作不创建真实报名、预约、票券或容量占用。

点击 Finish Route，完成页应显示地点 1/1、活动 1/1。

### 2:15–2:40｜双语与离线容灾

切换英文，说明语言只决定展示语言，不改变 `scenario_key` 或策展内容。

阻断 API 后再次提交，展示“Offline demo · Using the same-version local community catalog”；强调路线、引用和四条解释与在线一致。

### 2:40–3:00｜产品边界与未来形态

说明产品运行时无 AI/LLM；创新来自把有限的新居民需求组合成可验证、可解释、可离线执行的社区策展目录。本次评审主入口是 H5，最终仍可用同一 uni-app 代码发布微信小程序。

## GUI 最低回归组合

1. 中文 / 社区服务 / 初来第一周 / 独居 / 无额外需求。
2. 英文 / 亲子家庭 / 初来第一个月 / 有孩子 / 行动较慢。
3. 中文 / 餐饮 / 已稳定居住 / 伴侣 / 安静环境。
4. 英文 / 社交 / 初来第一周 / 合租 / 轮椅需求。

每组检查 120 分钟、四条理由、地点与活动引用、Demo Confirm、本地完成状态、无 raw enum、无误导设施承诺。
